Refactoring Plan: Profiler Report Generation

1. Separate Analyzing and Report Printing
   - Create distinct structures (e.g. `Analyzer` and `ReportPrinter`) to cleanly decouple the processing of profile entries from their terminal representation.

2. Path Formatting and Compacting
   - Render span paths in the format `parent/parent/span`.
   - For deeply nested spans (compact views), display only the first and last parents with the number of hidden parents: `parent/{depth}/parent/span`.
   - Implement string truncation for excessively long span names (showing only the start or end).

3. Enhanced Snapshotting
   - Ensure the snapshot mechanism captures and saves not only the analyzed results but also the raw input metrics.

4. Flat Source Snapshot Format
   - Avoid hierarchical / tree-like JSON formatting for source snapshots.
   - Store events in a compact, flat form (e.g. tracking the `published` vector of paths and results similarly to what's gathered during report creation).
